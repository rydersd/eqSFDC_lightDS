Qualify and Convert a New Lead

When a new lead arrives from a tradeshow, I want to assess their fit and intent quickly so I can either convert them or nurture them without wasting pipeline cycles.

Key Steps

  • Review lead score breakdown (engagement + fit + behavior)

Project Context

Equinix is standardizing its Salesforce Lightning experience across a global sales organization. The current state includes 800+ custom objects (consolidated to ~160), pages with 175-225 field sections and 190+ action buttons, and a swivel-chair workflow between Salesforce, Siebel, and email.

This prototype defines the target assembly pattern: how standard Salesforce Lightning components should be composed into consistent, persona-aware record pages. The three-column layout (Related | Details | Activity) serves as the foundation, with Agentforce touchpoints woven into the right-column Activity tabs.

The Monty Python data story keeps stakeholder conversations about content, layout, and field groupings engaging and low-stakes — so people focus on the structure, not the data.

Object Classification Methodology

Primary — Core deal lifecycle objects that every CRO persona interacts with daily. These get full record page prototypes with the three-column assembly pattern.

Secondary — Objects that support primary objects. They appear as related lists, tabs, or embedded components within primary record pages. They don't typically need their own standalone page layouts in this project.

Tertiary — Reference and admin objects. Important for configuration and governance but not part of the daily CRO workflow. Documented for completeness but not prototyped.
